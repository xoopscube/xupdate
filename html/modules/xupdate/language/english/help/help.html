Store List

The X-Update Store is a digital marketplace to get and deploy modules, preloads and themes. Trusted developer stores allow website owners to download and install add-ons and updates.

X-Update automatically monitors the Store public repository for updates as well as installed modules and themes.

Available updates are highlighted in the top notification, and the front-end if X-Update's block is installed.

If your SSL certificate is not up-to-date, from preferences check cURL: Disable SSL verify

Download and Install Updates

By default, X-Update compares the Version number of the currently installed add-on to the version number of the add-on in the public Store.

If there is an update available, then X-Update shows a notification.

NOTE ! You are given the freedom to use older versions if those are necessary for compatibility reasons.

Module

Modules can add new functionality to your Web Application Platform or extend and enhance existing functionality.

There are three different types of modules that are easily recognizable:

  • Core ( red label required )
  • Trust module ( duplicable )
  • X2 Module ( backwards compatibility )

Core modules

Required modules are installed by default along with recommended modules.

Trust modules

D3 nodules are installed in the Trust_PATH directory. These modules are duplicable and allow the installation of several instances on the client side. To rename the module directory :

  • 1. Hover over the module's name to check if "allowed".
  • 2. Type a new name for the module directory.
  • 3. Click "Rename Trust modules" at the bottom of the list.
  • 4. Confirm the new directory name.
  • 5. Proceed to install.

X2 Backwards Compatibility

The core modules, legacy System and Renderer, are designed to ensure backward compatibility and easier upgrades from previous versions, to be forward compatible and extensible, with built-in features such as hooks, extensions (preloads), and an interface that allows the addition of new features.

NOTE ! The Legacy system can not guarantee backwards compatibility with deprecated code of module versions that are no longer supported by developers.

Theme

This section provides a brief introduction to download and install Themes.

Select a Theme to customize the global appearance of your website.

Browse to System ¬ª Theme Management

Theme Management allows you to simple click the "Select" button next to the theme that you want to use as the default theme or main theme of your site.

Allow Users to select Theme

Any theme that is "checked" in the "Control" column, will be available to users in the block "Themes".

If you install the "Themes" block and grant users permission, they can then select their favorite theme and customize the look and feel of the website.

ThemeFinder

Service to select and preview Themes.

Note : The service is not up-to-date and might be removed soon due the lack of developers interest.

Preload

This section provides a brief introduction to Preloads and walks through the installation of a simple extension.

Preloads are commonly Single File Components that extend the Web Application Platform with customized functionality.

  • ‚Äî Productivity workflow
  • ‚Äî Web page content enrichment
  • ‚Äî Information aggregation
  • ‚Äî UI components

In a nutshell, Preloads are simple extensions, or plugins, to add new functionality to a website or improve the current usability of a design without hacking the core system.

Enable and disable a Preload

Unlike modules, Preloads do not have a dashboard control panel.

A Preload is installed in the public directory /preloads/...

To disable a preload, the best practice is to use XelFinder to move the preload to the directory /preloads/disabled/ or simply delete.
